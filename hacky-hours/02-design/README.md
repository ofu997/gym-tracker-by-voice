# Design Documents

This folder contains design documents that define how the product works.
A new collaborator should be able to read these and understand the product well enough to build it.

## Documents

| File | Purpose | Status |
|------|---------|--------|
| ARCHITECTURE.md | System shape, stack decisions, component overview | done |
| DATA_MODEL.md | What gets stored and how | done |
| BUSINESS_LOGIC.md | Parsing rules, progression calculation, digest | done |
| USER_JOURNEYS.md | How users move through each input modality | done |
| SECURITY_PRIVACY.md | Data flow, third-party disclosure, trust boundaries | done |
| LICENSING.md | License choice and dependency constraints | done |
| ACCESSIBILITY.md | Accessibility standards and commitments | done |
| decisions/ | Architecture Decision Records (ADRs) | empty |

Add documents here as needed. Not every project needs all of them.
See `/hacky-hours help design` for the full list of available doc types.
